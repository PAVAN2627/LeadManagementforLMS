# Plan 3.2: Automated Email Onboarding via Brevo

## Objective
Integrate the Brevo API to trigger automated welcome emails when an Admin creates a generic new user, completing REQ-08.

## Context
Load these files for context:
- .gsd/SPEC.md
- .gsd/REQUIREMENTS.md
- api/users/index.ts
- .env

## Tasks

<task type="auto">
  <name>Update Environment Variables for Brevo</name>
  <files>
    .env
  </files>
  <action>
    Ensure `BREVO_API_KEY` is loaded and validated via `dotenv`. The key is an API key (not SMTP), as set in `.env`.
  </action>
  <verify>grep BREVO_API_KEY .env</verify>
  <done>Brevo API Key available in the environment.</done>
</task>

<task type="auto">
  <name>Integrate Email Trigger in User Creation</name>
  <files>
    api/users/index.ts
    src/lib/email.ts
  </files>
  <action>
    Create an email service utility (`src/lib/email.ts`) using the Brevo API, using native `fetch` or a similar HTTP client instead of an SDK.
    When a `POST` request to `/api/users` is successfully completed, send an email to the new user's email address providing a welcome message and onboarding instructions.
    
    API Endpoint: `POST https://api.brevo.com/v3/emailCampaigns` (or appropriate endpoint for transactional emails if preferred over campaigns). 
    Headers: `api-key: YOUR_API_V3_KEY`
    
    Reference Payload for API:
    ```json
    {
      "name": "Campaign sent via the API",
      "subject": "Welcome to Lead Management System",
      "sender": { "name": "Admin", "email": "admin@yourcompany.com" },
      "type": "classic",
      "htmlContent": "Congratulations! You successfully gained access to the system.",
      "recipients": { "listIds": [2,7] } // or targeting specific emails natively depending on transactional/campaign api choice
    }
    ```
    (Note: Even though the user example shows `emailCampaigns`, you might need to use `https://api.brevo.com/v3/smtp/email` for individual transactional emails to new users. Adapt payload suitably: `{"to": [{"email": "newuser@example.com"}], "subject": "Welcome", ...}`)

    Do not block the HTTP response if the email fails, but log the error appropriately.
  </action>
  <verify>Attempt creating a user while mocking out the email send function.</verify>
  <done>Email onboarding triggered successfully on User Creation.</done>
</task>

## Success Criteria
- [x] New agents/managers receive a welcome email with system instructions.
- [x] Email delivery failure does not crash the overall user creation flow.
